# 💡 Top 3 改进建议

### 1. 🔴 最关键：消灭"假数据降级"反模式（影响信任底线）

**问题：** Skill 最危险的设计模式不是报错，而是**返回看起来像正确数据的垃圾数据**。

```
用户看到：pronunciation = "/infrastructure/"    ← 看起来像音标
实际上：这只是单词本身加了斜杠                  ← 完全是假数据

用户看到：definition = "Word from reading text"   ← 看起来像释义
实际上：这是硬编码的固定字符串                   ← 完全是假数据
```

**在哪改：** `src/tts_handler.py:46,61` + `src/tts_handler.py:153-170` + `src/ai_analysis.py:104`
**怎么改：**
- IPA 失败 → 返回 `"N/A"` 或调用 AI fallback → **禁止返回假音标**
- Definition 缺失 → 返回空字符串 + `"[待补充]"` 标记 → **禁止硬编码占位符**
- API 不可用 → 返回带 `error_flag` 的结构化结果 → **禁止静默吞异常**

**预期效果：** 用户至少能分辨"哪些结果可靠、哪些暂时不可用"，而不是被假数据误导。

---

### 2. 🔴 次关键：修 SSRF 漏洞 + 补齐 SKILL.md 核心骨架（影响上线资格）

**SSRF 修复（安全底线）：**

**在哪改：** `src/parser.py:42-74`
**怎么改：** 添加 `validate_url()` 函数：
```python
import ipaddress, urllib.parse

def validate_url(url: str) -> bool:
    parsed = urllib.parse.urlparse(url)
    if parsed.scheme != 'https':  # 仅允许 https
        raise ValueError("仅支持 HTTPS 协议")
    try:
        ip = ipaddress.ip_address(parsed.hostname)
        if ip.is_private or ip.is_loopback or ip.is_reserved:
            raise ValueError("不允许访问内网地址")
    except ValueError:
        pass  # 域名形式，后续可加白名单
    return True
```

**SKILL.md 骨架补齐（产品化底线）：**

至少补齐 **6 项高优先缺失**（R-L1-01 ~ R-L1-06）：角色定义 / 能力边界 / 输出模板 / 行为规则 / 合规模块 / 开场白。预估增加 SKILL.md 到 **150-200 行**，从 0.95 分提升到 **55-65 分**区间。

**预期效果：** 安全漏洞清零 + 配方从 F 提升到 D-/D，综合总分从 48.8 → **~58-62**（接近及格线）。

### 3. 🟡 建议优化：建立 fallback 链 + 性能止血（影响用户体验）

**词汇分类 fallback 链：**

**在哪改：** `src/nlp_analyzer.py:92-107`
**怎么改：**
```
当前：set lookup → miss → "unknown"（一步到位，42.5% 误分类）
改为：
```

**预期效果：** 词汇分类准确率从 57.5% → 85%+；典型场景端到端耗时从 35-60s → 20-30s。
